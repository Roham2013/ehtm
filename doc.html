Getting Started / Introduction

Introduction to EHTM

EHTM (Easy HyperText Markup Language) is a simple yet powerful markup language that compiles to standard HTML. It ships as a single standalone executable (ehtm.exe) requiring no external dependencies.

Standalone Executable

Single ehtm.exe file. No Python, no pip, no dependencies required.

Clean Syntax

No closing tags needed. Use indentation to define structure, just like Python.

Variables

Define reusable values once and use them throughout your templates.

Components

Include other files with the Include directive for modular code.

Optimized Output

Generate minified or pretty-printed HTML with a single command.

Cross-Platform

Runs on Windows, Linux, and macOS as a native executable.

Why EHTM?
Writing raw HTML can be tedious and repetitive. EHTM eliminates closing tags and uses indentation-based structure, making coding up to 50% faster. As a compiled executable, it requires zero setup -- just download and run.
HTML vs EHTM Comparison
Standard HTML
HTML <div id="main" class="container"> <h1>Hello World</h1> <p>This is a paragraph.</p> </div>
EHTM
EHTM div#main.container h1 "Hello World" p "This is a paragraph."

Installation

EHTM is distributed as a single standalone executable (ehtm.exe). No Python, no package manager, no dependencies are needed.

1

Download the Executable

Download ehtm.exe and place it in your project directory or any folder included in your system's PATH.

2

Verify Installation

Open a terminal and run:

CMD ehtm --version # EHTM Compiler v3.0
3

Create Your First File

Create a new file with the .ehtm extension and start coding.

4

Run the Tests (Optional)

Verify everything works correctly:

CMD ehtm --test
READY You are now ready to use EHTM. Create your first .ehtm file and compile it with ehtm input.ehtm.

Quick Start

Build your first web page with EHTM in less than 2 minutes.

Step 1: Create Your EHTM File
Create a new file named hello.ehtm with the following content:
EHTM html lang="en" head meta charset="UTF-8" title "My First Page" body h1 "Hello EHTM!" p "This is my first EHTM page."
Step 2: Compile to HTML
Run the following command in your terminal:
CMD ehtm hello.ehtm
This generates hello.html which you can open in any browser.
Step 3: Customize Output
Generate minified output for production:
CMD ehtm hello.ehtm -f minified -o dist/index.html

Elements & Tags

In EHTM, elements start with a tag name and their content is defined by indentation level.

Nested Elements
Parent-child relationships are determined by indentation (spaces):
EHTM div h1 "Page Title" p "Paragraph inside div" ul li "Item 1" li "Item 2"
Void (Self-Closing) Elements
Tags like br, hr, img, input are automatically recognized and rendered without closing tags:
EHTM br hr img src="photo.jpg" alt="A photo" input type="text" name="username"
Generated HTML: <br>, <hr>, <img src="photo.jpg" alt="A photo">

Attributes

Attributes can be defined in two ways: space-separated or inside parentheses.

Method 1: Space-Separated
Write attributes with spaces before the text content:
EHTM a href="/about" target="_blank" "About Us" input type="email" placeholder="Your email" required
Method 2: Parenthesized (Comma-Separated)
Use parentheses and commas for cleaner multi-attribute elements:
EHTM button (type="submit", class="btn btn-primary", disabled) "Submit"
NOTE Boolean attributes (like disabled, required, checked) are automatically detected when no value is provided.

IDs & Classes

Use CSS-like syntax: # for ID and . for class.

EHTM div#main.container.highlight "Content" # Generates: <div id="main" class="container highlight">Content</div> section#hero.full-width h1 "Hero Title" p.lead "Hero description"

Text Content

Text is placed inside quotes " " or ' ' after the tag and its attributes.

EHTM h1 "This is a heading" p "This is a paragraph with <strong>bold text</strong>." a href="/link" "Click here"
IMPORTANT If your text contains quotes, escape them with \: "He said \"Hello\"". Alternatively, use single quotes for the wrapper: 'He said "Hello"'.

Variables

Define reusable values with the set keyword and reference them using ${variable_name}.

EHTM set page_title = "My Website" set year = "2024" html head title "${page_title}" body h1 "Welcome to ${page_title}" footer p "© ${year} All rights reserved."

Components (Include)

Use the include directive to import other EHTM files for component-based architecture.

EHTM # main.ehtm html head include "components/head.ehtm" body include "components/header.ehtm" main h1 "Page Content" include "components/footer.ehtm"
WARNING Circular includes are detected and will raise an error. Included files share the parent's variables.

Comments

Comments start with # and extend to the end of the line.

EHTM # This is a comment # TODO: Add navigation items div.container h1 "Title" # Main content section p "Content goes here"

Pretty Output

The pretty format generates well-indented, human-readable HTML.

CMD ehtm input.ehtm -f pretty -i 4
OptionDescriptionDefault
-f prettyEnable pretty output formatEnabled
-i <number>Indentation size in spaces4

Minified Output

The minified format removes all unnecessary whitespace for production.

CMD ehtm input.ehtm -f minified -o dist/output.min.html
PRO TIP Use --no-doctype if injecting the output into an existing template.

Watch Mode

Monitor a directory and automatically recompile on changes.

CMD ehtm -w ./templates
Press Ctrl+C to stop watching. Combine with -f pretty for readable output during development.

Common Errors

Troubleshooting guide for frequent compilation errors.

Undefined Variable
Error: Undefined variable: 'title'. Did you forget to declare it with 'set'?
Solution: Add set title = "value" at the top of your file.
Inconsistent Indentation
Error: Inconsistent indentation level
Solution: Use consistent spacing. 4 spaces per level recommended. Do not mix tabs and spaces.
File Not Found
Error: Included file not found: 'header.ehtm'
Solution: Check the path is correct and relative to the current file.
'ehtm' is not recognized
Error: 'ehtm' is not recognized as an internal or external command
Solution: Add the folder containing ehtm.exe to your system PATH, or use the full path: C:\tools\ehtm.exe input.ehtm.

Frequently Asked Questions

Answers to common questions about EHTM.

Do I need Python installed to use EHTM?
No. EHTM is distributed as a standalone executable (ehtm.exe). It includes all necessary components and requires no runtime dependencies.
Can I use EHTM on Windows, Mac, and Linux?
Yes. Platform-specific executables are available for Windows (.exe), macOS, and Linux. All versions have identical functionality.
How do I update EHTM?
Simply download the latest ehtm.exe and replace the old file. Your .ehtm source files remain unchanged and compatible.
Can I use JavaScript and CSS with EHTM?
Yes. EHTM generates standard HTML. Include <script> and <link> tags as usual. Inline styles work identically to HTML.
Is EHTM production-ready?
Yes. The compiler generates clean, standards-compliant HTML. Use the minified format for production deployments. Always validate generated HTML before deploying.
